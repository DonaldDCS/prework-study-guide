HTML

  • The head element contains information (meta) about the webpage
  • The body element represents the visible content shown to the user
  • Element "h1" though "h6" should be used in a logical order to assist screen readers and "h1" should only appear once on a page
  • Element section is self-explanatory
  • Element definitions for list: "ul" = unorder list, "ol" = ordered list, "li" = list item
  • Element "class" tag defines the class to use from css file(s)
  • Element "id" tag gives the element a unique ID and should never be shared with other elements

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Order of Commands - Starting a new branch

  1. Use "cd" to navigate to correct directory
  2. Use "git status" to check status and location
  3. Use "git checkout main" to ensure on main branch (if branching from there)
  4. Use "git pull origin main" to ensure local and remote main branches are synced
  5. Use "git checkout -b -branch name-" to create a new branch (example: feature/add-css)
  6. Navigate to correct directory using "cd" and use "touch -filename-" if you need to create something
